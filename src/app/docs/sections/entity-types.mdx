## Entity Types

<span id="entity-types" />

Person records include `birth_date` and `death_date` fields. The `is_historic` property returns `True` for deceased individuals. A single person can have multiple records with different role/org combinations (unique on `source_id + role + org`).

### Location Types

<span id="location-types" />

Locations use a two-level type system: a detailed `location_type` string (from Wikidata, e.g. `us_state`, `sovereign_state`) and a `simplified_type` enum for easy filtering.

<table>
  <thead>
    <tr>
      <th>SimplifiedLocationType</th>
      <th>Description</th>
      <th>Examples</th>
    </tr>
  </thead>
  <tbody>
    <tr><td><code>continent</code></td><td>Continents</td><td>Europe, Asia, Africa</td></tr>
    <tr><td><code>country</code></td><td>Countries and sovereign states</td><td>United States, Germany, Japan</td></tr>
    <tr><td><code>subdivision</code></td><td>States, provinces, regions</td><td>California, Bavaria, Ontario</td></tr>
    <tr><td><code>city</code></td><td>Cities, towns, municipalities</td><td>New York, London, Tokyo</td></tr>
    <tr><td><code>district</code></td><td>Districts, boroughs</td><td>Manhattan, Westminster</td></tr>
    <tr><td><code>historic</code></td><td>Former countries, historic territories</td><td>Soviet Union, Czechoslovakia</td></tr>
    <tr><td><code>other</code></td><td>Unclassified</td><td>Miscellaneous locations</td></tr>
  </tbody>
</table>

Locations include hierarchical `parent_ids` for navigating the geographic hierarchy (e.g., a city points to its state, which points to its country).

### Source Types

<span id="source-types" />

Every record tracks its data provenance:

<table>
  <thead>
    <tr>
      <th>Source</th>
      <th>Entity Types</th>
      <th>Identifier Format</th>
    </tr>
  </thead>
  <tbody>
    <tr><td><code>gleif</code></td><td>Organizations</td><td>LEI (20-char alphanumeric, e.g. <code>549300XYZ...</code>)</td></tr>
    <tr><td><code>sec_edgar</code></td><td>Organizations, People</td><td>CIK (integer, e.g. <code>789019</code>)</td></tr>
    <tr><td><code>companies_house</code></td><td>Organizations, People</td><td>Company number (e.g. <code>01624297</code>)</td></tr>
    <tr><td><code>wikidata</code></td><td>Organizations, People, Roles, Locations</td><td>QID (e.g. <code>Q312</code>)</td></tr>
    <tr><td><code>sec_form4</code></td><td>People</td><td>CIK of the reporting person</td></tr>
    <tr><td><code>companies_house_officers</code></td><td>People</td><td>Officer ID from Companies House</td></tr>
  </tbody>
</table>

Records from different sources can be linked via **canonicalization**, which identifies equivalent entities across sources and stores the link in the `canonical_id` foreign key column.
